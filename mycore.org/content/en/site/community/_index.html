MyCoRe

MyCoRe is an open source project, that derived from the MILESS Project of the University Duisburg-Essen. An association of developers from various German universities and universities library are jointly developing the framework. This team is the MyCoRe Developer Community.

How is the community organized?

As a professional open source project with a great deal of responsibility for the applications created, it is helpful to use appropriate tools for team-driven development. The following figure shows a overview of the organizational structure and the tools used.

The MyCoRe Office

The MyCoRe developer community has decided to improve communication with the users, to coordinate the work and the external presentation of the project, a business office to set up. This should be a first port of call for all interested parties.

The duties of the office include the organization of developer meetings and the user workshop. How to reach the office is explained below Contact.

Communication with the developers

You can reach us under the mailing lists. Beside that there are additional Information for users in a Wiki, with How to? instructions. Error messages can be sent directly to our ticket system Atlassian JIRA.

Tools for development of MyCoRe

In detail, the following software is used:

  • As a developer platform Eclipse is used.
  • The MyCoRe code is hosted and versioned on GitHub.
  • If changes and further developments are committed to the GitHub system, the comments are automatically transferred to the corresponding ticket. Here we are using Atlassian JIRA.
  • In cyclic intervals, Atlassian Bamboo integration test are performed by Atlassian Bamboo.
  • Successful builds of snapshots are verified by Travis and Codacy, than the code is delivered to Sonatype MyCoRe-Artifactory, where it is provided for delivery via Maven. In analogy to this releases are provided in Maven-Central.
