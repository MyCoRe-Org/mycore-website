Konfiguration

Das Überschreiben von Ressourcen funktioniert für folgende Ressource-Typen:

  • XSL-Stylesheets
  • XEditor-Formulare
  • Sass-Stylesheets

Durch das Property MCR.Developer.Resource.Override wird bestimmt woher die Resourcen zum Überschreiben geladen werden. Die Pfade die als erstes aufgelistet werden haben eine höhere Priorität.
Damit das Überschreiben von XSL-Stylesheets funktioniert, muss außerdem der XSL-Template-Cache ausgeschaltet werden.
In der MyCoRe-Standardkonfiguration werden Sass-Stylesheets nur einmal in CSS übersetzt und das Ergebnis gespeichert. Um eine permanente CSS-Übersetzung zu erzwingen muss man den Sass-Entwickler-Modus einschalten.

Zusammengefasst sieht eine solche Konfiguration z.B. für die Viewer- und MIR-Entwicklung wie folgt aus: {{< highlight shell "linenos=table">}} MCR.Developer.Resource.Override=/Pfad-zu-mir/mir-layout/src/main/resources,/Pfad-zu-mycore/mycore-viewer/src/main/resources MCR.LayoutService.LastModifiedCheckPeriod=0 MCR.UseXSLTemplateCache=false MCR.SASS.DeveloperMode=true{{< /highlight >}}

Das Überschreiben einer Resource wird beim Abrufen mit einer Log-Meldung quittiert {{< highlight shell "linenos=table">}} Found overridden Resource: /Pfad-zu-mir/mir-layout/src/main/resources/{{< /highlight >}}
