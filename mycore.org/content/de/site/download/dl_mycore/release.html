Informationen zur Migration vom LTS 2019.06 finden Sie im Abschnitt zur Migration.

Was ist neu?

  • Die „REST-API v2“ wurde finalisiert, dokumentiert und kann nun parallel zur ersten Version genutzt werden. Der REST-Client ist jetzt ein Projekt auf GitHub.
  • Der Umstieg auf XSLT, Version 3.0 (XSLT v3) wurde begonnen. Dazu haben sich die Entwicklerinnen und Entwickler teilweise bereits in XSLT v3 eingearbeitet. Nun sollen alte XSL-Stylesheets schrittweise umgearbeitet und neue direkt in dem Format umgesetzt werden.
  • Es gab Neuerungen rund um Solr mit einem Upgrade auf Solr Version 8.4: Durch den Wechsel auf Solr v8 können wir einen Workaround für die Behandlung von verschachtelten Index-Dokumenten aus MyCoRe entfernen. Dies wird nun von Solr selbst korrekt realisiert.
  • Im Rahmen der Umstellung auf Solr v8 konnte außerdem die Anbindung optimiert werden, so dass u.a. nun wieder eine parallele und damit performantere Indizierung möglich ist.
  • Weiter gibt es Neues zum Thema Internationalisierung: Ein neuer i18n-Mechanismus wurde entwickelt, um einheitlich in Webseiten, Formularen und XSL Stylesheets sprachabhängige Ausgaben definieren zu können. Dieser kann parallel zu den bereits bestehenden Mechanismen eingesetzt werden.
  • Ein Transformer wurde entwickelt, mit dessen Hilfe Teile statisch vorgeneriert und per URI-Resolver eingebunden werden können (MCRStaticContentGenerator). Dies wird z.B. verwendet um die Versionshistorie von Metadaten als HTML-Schnipsel bereitzustellen.
  • Mit der neuen MyCoRe-Komponente „mycore-csl“ (Citation Style Language) können Objekte in verschiedene Zitationsstile transformiert werden. Damit ist es für das „Metadata Object Description Schema“ (MODS) möglich, alle in CSL angebotenen Zitierstile auszu­geben.
  • Es gibt Neuerungen im Modul zum Generieren der „Persistent Identifier“ (PI-Modul), so können z.B. Regeln definiert werden, die gegebenenfalls dazu führen, dass automatisch ein oder mehrere Persistente IDs (PI) für das entsprechende MyCoRe-Objekt generiert werden (z.B. wenn ein bestimmter Dokumentstatus vorliegt, das Dokument vom Typ Artikel ist und noch keine DOI angegeben ist).
  • Auch ist es möglich xPath-Angaben in das PI-Schema mit aufzunehmen; bisher war es nur möglich die ID des MyCoRe-Objektes oder Systemdaten dafür zu nutzen. Nun können Metadaten wie beispielsweise der Dokumenttyp und/oder die zugeordnete Einrichtungs-ID Teil des PI-Schemas sein.
  • Das Rechtemanagement für die Derivate (zum Objekt zugehörige Dateicontainer) wurde überarbeitet, vereinheitlicht und entsprechend dokumentiert. Es gibt nun mit read, view und preview drei Permissions, die festlegen ob Dateien innerhalb eines Containers gelesen, nur angeschaut (z.B. im MyCoRe-Viewer) oder nur ein definiertes Vorschaubild angezeigt werden darf.
  • Es wurde entschieden, dass die Unterstützung für den Microsoft Internet Explorer (IE) mittelfristig eingestellt werden soll. Dafür wird beispielsweise in MIR-Anwendungen standardmäßig eine entsprechende Meldung ausgegeben, wenn die Seiten mit dem IE auf­gerufen werden. Die Community hat beschlossen, dass die Statistiken der Anwendungen regelmäßig geprüft werden und wenn die IE-Nutzung deutlich zurück gegangen ist, soll die Unterstützung eingestellt werden.

Einbindung der Komponenten

Die Einbindung der Komponenten aus dem MyCoRe-Kern erfolgt über Maven und die entsprechenden, für die eigene Anwendung erforderlichen Abhängigkeiten. Um das LTS 2020.06 in der eigenen Anwendung zu verwenden, müssen in der pom.xml die MyCoRe-Version und mycore-parent angepasst werden:

{{< highlight xml "linenos=table">}} ... org.mycore mycore-parent 46 ... 2020.06.3-SNAPSHOT ... org.mycore mycore-meta ${mycore.version} ... ... {{< /highlight >}}
